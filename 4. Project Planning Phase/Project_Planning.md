# Phase 4: Project Planning Phase

This document defines the project scheduling, milestones, and task allocations for the **Rising Waters** application.

---

## 1. Project Timeline & Milestones

The project was executed across four structured sprints:

| Sprint | Phase | Duration | Core Deliverables |
| :--- | :--- | :--- | :--- |
| **Sprint 1** | Brainstorming & Requirement Analysis | Week 1 | Problem statement definition, target dataset schema analysis, and hardware/software setup. |
| **Sprint 2** | Preprocessing & Pipeline Design | Week 2 | IQR outlier logic implementation, dataset imputation, model training (DT, RF, KNN, XGBoost) and serialization scripts. |
| **Sprint 3** | Backend & Front-End Development | Week 3 | Flask app routing, prediction logging in local CSV, client-side validations, and jsPDF exporter integration. |
| **Sprint 4** | System Integration & Testing | Week 4 | Error handling (404/500), model performance graphing, visual mockup creation, and Git repository restructuring. |
